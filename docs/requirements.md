# Требования

## Функциональные требования
### FR-1: Пользователь может запускать приложение в двух режимах:
* сервер (ожидает подключения)
* клиент (подключается к peer)
### FR-2: Пользователь задаёт:
* имя пользователя
* адрес и порт peer (опционально)
### FR-3: Пользователь может отправлять текстовые сообщения
### FR-4: Пользователь получает сообщения от peer
### FR-5: Каждое сообщение содержит:
* имя отправителя
* timestamp
* текст
### FR-6: Соединение устанавливается напрямую между узлами


## Нефункциональные требования
### NFR-1: Низкая задержка 
### NFR-2: Простота запуска (CLI)
### NFR-3: Потокобезопасность
### NFR-4: Расширяемость (например, добавление шифрования)
